🌟 Introducción al mundo de CSS

Aprende los fundamentos para dar estilo a tus páginas web

¿Qué es CSS?

CSS (Cascading Style Sheets) es un lenguaje que define la presentación de un documento escrito en HTML. Con CSS controlamos colores, fuentes, márgenes, posiciones y mucho más.

CSS funciona mediante reglas que asocian propiedades y valores a elementos HTML.

Sintaxis básica

Una regla CSS se compone de un selector y un bloque de declaraciones entre llaves { }:

selector {
  propiedad: valor;
  otra-propiedad: otro-valor;
}

Por ejemplo:

p {
  color: blue;
  font-size: 16px;
}

Esto haría que todos los párrafos <p> tengan texto azul y tamaño 16 píxeles.

Formas de aplicar CSS

  • CSS en línea (inline): mediante el atributo style en una etiqueta HTML.
    <p style="color: red;">Texto rojo</p>
  • CSS interno: dentro de una etiqueta <style> en el <head> del documento.
  • CSS externo: en un archivo separado con extensión .css y enlazado con <link>.

Esta misma página usa CSS interno (mira dentro de <style> en el código fuente).

Selectores comunes

  • Selector de elemento: p, h1, div → afecta a todas las etiquetas de ese tipo.
  • Selector de clase: .nombre-clase → afecta a elementos con class="nombre-clase".
  • Selector de ID: #id → afecta al elemento con id="id" (debe ser único).

Ejemplo: En esta página usamos la clase .ejemplo para los recuadros grises.

Propiedades básicas

Algunas propiedades que puedes usar:

  • color: color del texto.
  • background-color: color de fondo.
  • font-size: tamaño de la fuente.
  • margin: margen exterior.
  • padding: relleno interior.
  • border: borde.
h1 {
  color: white;
  background-color: #4CAF50;
  padding: 10px;
  border-radius: 5px;
}

¿Qué es el Box Model?

El Box Model es un concepto fundamental en CSS que describe cómo se estructuran y representan los elementos en la página web. Cada elemento se representa como una caja compuesta por: contenido, padding (relleno), borde y margen.

Margen
Borde
Relleno
Contenido
  • Contenido: Lo que ves dentro de la caja (texto, imagen, etc).
  • Padding (Relleno): Espacio entre el contenido y el borde.
  • Borde: Línea que rodea el padding y el contenido.
  • Margen: Espacio fuera del borde, separa la caja de otros elementos.

¿Qué es box-sizing?

La propiedad box-sizing en CSS define cómo se calculan el ancho y alto total de los elementos. Por defecto, el tamaño de un elemento se calcula sumando el contenido, el padding y el borde (content-box). Si usas box-sizing: border-box, el padding y el borde se incluyen dentro del ancho y alto especificados, facilitando el diseño de layouts.

content-box (por defecto)
width: 200px;
padding: 20px;
border: 4px solid #2196f3;
border-box
width: 200px;
padding: 20px;
border: 4px solid #4caf50;
box-sizing: border-box;
  • content-box: El ancho no incluye padding ni borde. El elemento será más grande de lo que defines.
  • border-box: El ancho sí incluye padding y borde. El tamaño total es exactamente el que defines.

Herencia de estilos en CSS

En CSS, muchas propiedades se heredan automáticamente de los elementos padres a sus hijos. Esto significa que si defines un estilo en un contenedor, sus elementos internos pueden recibir ese mismo estilo sin necesidad de volver a declararlo.

Div padre (color: purple; font-family: monospace;)

Este párrafo hereda el color y la fuente del padre.

  • Este texto también hereda.
  • ¿Qué se hereda? Propiedades como color, font-family, font-size, line-height, entre otras.
  • ¿Qué no se hereda? Propiedades como margin, padding, border, background no se heredan por defecto.
  • Utilidad: La herencia permite escribir menos código y mantener estilos consistentes.
  • Característica: Puedes forzar la herencia usando inherit o evitarla con initial o unset.
.padre {
  color: purple;
  font-family: monospace;
}

Especificidad y fuentes en CSS

Especificidad determina qué regla CSS se aplica cuando varias coinciden sobre el mismo elemento. Las reglas más específicas tienen prioridad sobre las menos específicas.

¡Texto de ejemplo!

  • Selector de elemento (p): especificidad baja.
  • Selector de clase (.importante): especificidad media.
  • Selector de ID (#demo): especificidad alta.
  • Si hay conflicto, gana la regla con mayor especificidad.
p { color: gray; }
.importante { color: orange; }
#demo { color: red; }

Fuentes en CSS permiten personalizar la tipografía de tu sitio. Puedes usar fuentes estándar, importar fuentes externas o personalizadas.

Esta línea usa 'Courier New'.

  • Utilidad: La especificidad ayuda a controlar qué estilos se aplican. Las fuentes mejoran la identidad visual y legibilidad.
  • Características: Puedes combinar fuentes, usar font-family, importar fuentes con @import o @font-face.
.fuente1 { font-family: Arial, sans-serif; }
.fuente2 { font-family: 'Courier New', monospace; }

Introducción a Flexbox

Flexbox (Flexible Box Layout) es un modelo de diseño en CSS que facilita la creación de layouts flexibles y eficientes, permitiendo alinear y distribuir espacio entre los elementos de un contenedor, incluso cuando su tamaño es desconocido o dinámico.

  • ¿Para qué sirve? Para crear diseños responsivos, centrar elementos fácilmente, distribuir espacio y controlar el alineamiento de los hijos de un contenedor.
  • ¿Cómo se usa? Se activa con display: flex; en el contenedor. Luego, puedes usar propiedades como justify-content, align-items y flex-direction para controlar la disposición de los elementos.

Introducción a position: static, relative y absolute

La propiedad position en CSS controla cómo se posiciona un elemento en la página. Los valores más comunes son:

  • static (por defecto): El elemento sigue el flujo normal del documento. No se puede mover con top, left, right o bottom.
  • relative: El elemento se posiciona relativo a su posición original. Puedes moverlo usando top, left, etc., pero sigue ocupando su espacio original.
  • absolute: El elemento se posiciona respecto a su primer ancestro posicionado (que no sea static) o al viewport. Sale del flujo normal y no ocupa espacio.

Introducción a position: fixed y sticky

position: fixed y position: sticky son valores avanzados de la propiedad position en CSS que permiten crear elementos que permanecen visibles en la pantalla de formas especiales.

  • fixed: El elemento se posiciona respecto a la ventana del navegador (viewport). Permanece fijo aunque hagas scroll. Se usa para menús, botones flotantes, banners, etc.
  • sticky: El elemento actúa como relative hasta que alcanza un punto definido (por ejemplo, top: 0), y luego se comporta como fixed mientras se hace scroll. Es útil para encabezados o menús que deben quedarse visibles al hacer scroll.

.fixed-ejemplo {
  position: fixed;
  top: 20px;
  right: 20px;
  background: #ffccbc;
  padding: 12px 20px;
  border-radius: 6px;
}
.sticky-ejemplo {
  position: sticky;
  top: 0;
  background: #dcedc8;
  padding: 12px 20px;
  border-radius: 6px;
}
  • Características de fixed: Siempre visible, no depende del scroll del contenido, puede superponerse a otros elementos.
  • Características de sticky: Solo funciona dentro de su contenedor, necesita un valor de top, left, etc., y requiere suficiente espacio en el contenedor para activarse.

Alineación con Flexbox

Flexbox no solo sirve para distribuir elementos, sino que ofrece un potente sistema de alineación tanto en el eje principal (horizontal por defecto) como en el eje cruzado (vertical). Con las propiedades adecuadas podemos controlar la posición de los elementos de manera precisa y sencilla.

🎯 Alinear y centrar

Las propiedades justify-content y align-items permiten alinear los elementos hijos en el eje principal y cruzado respectivamente. Con ellas podemos centrar elementos de forma fácil, algo que antes requería trucos complicados.

¿Qué es? justify-content controla la alineación horizontal (si flex-direction: row) y align-items controla la alineación vertical. Valores comunes: center, flex-start, flex-end, space-between, space-around.

Usos: Centrar un botón en un contenedor, distribuir elementos de menú, alinear íconos con texto, etc.

1
2
3
.flexbox-demo {
  display: flex;
  justify-content: center; /* Centra horizontalmente */
  align-items: center; /* Centra verticalmente */
  height: 150px;
}
.flexbox-demo-item {
  background: #90caf9;
  padding: 16px 24px;
  border-radius: 6px;
}

📏 Alineación multilínea

Cuando los elementos no caben en una sola línea y usamos flex-wrap: wrap, podemos controlar cómo se alinean los grupos de elementos con align-content.

¿Qué es? align-content alinea las líneas completas dentro del contenedor cuando hay espacio sobrante en el eje cruzado. Funciona similar a justify-content pero entre líneas, no entre elementos.

Usos: Galerías de imágenes, grids de productos, layouts con muchas tarjetas que deben reorganizarse en filas.

.flexbox-multiline-item {

🔧 Alineaciones específicas

A veces necesitamos que un elemento en particular se alinee de manera diferente al resto. Para eso existe align-self, que permite sobrescribir la alineación definida por align-items en un elemento hijo concreto.

¿Qué es? align-self acepta los mismos valores que align-items (center, flex-start, flex-end, etc.) y se aplica directamente sobre el elemento que queremos personalizar.

Usos: Destacar un elemento, crear diseños asimétricos, ajustar la posición de un botón dentro de una tarjeta sin afectar a los demás.

1
2 (centrado)
3
.flexbox-specific {
  display: flex;
  align-items: flex-start; /* Por defecto, todos arriba */
  height: 150px;
  gap: 10px;
}
.flexbox-specific-item {
  background: #b39ddb;
  padding: 16px;
  border-radius: 6px;
}
.especial {
  align-self: center; /* Solo este se centra */
}

🧭 ¿Qué es una Navbar y para qué sirve?

Navbar (abreviatura de "navigation bar") es una barra de navegación que permite a los usuarios moverse entre las diferentes secciones o páginas de un sitio web. Es uno de los componentes más importantes en el diseño web, ya que proporciona la estructura de navegación principal.

📋 ¿Para qué sirve?

  • Organizar el contenido: Ayuda a estructurar la información del sitio en categorías lógicas.
  • Mejorar la experiencia de usuario: Los usuarios pueden encontrar rápidamente lo que buscan.
  • Consistencia: Aparece en todas las páginas, manteniendo la navegación familiar.
  • Jerarquía visual: Muestra la importancia de las diferentes secciones.
  • Responsive: Las navbars modernas se adaptan a diferentes tamaños de pantalla.

🎨 Tipos de Navbar

  • Navbar horizontal: La más común, con enlaces en una línea horizontal.
  • Navbar vertical/sidebar: Colocada en un lateral, útil para dashboards.
  • Navbar fija/sticky: Permanece visible al hacer scroll.
  • Navbar responsive: Se adapta a móviles con menú hamburguesa.
  • Navbar con dropdown: Incluye submenús desplegables.

💡 Buenas prácticas

  • Mantener los enlaces claros y concisos.
  • Usar máximo 5-7 elementos para no saturar.
  • Indicar la página actual con un estilo diferente (active).
  • Asegurar que sea accesible con teclado y lectores de pantalla.
  • Incluir el logo enlazado a la página principal.

🍪 ¿Qué es un Footer de Cookies y para qué sirve?

El footer de cookies (también conocido como "cookie banner", "cookie consent" o "aviso de cookies") es un elemento informativo que aparece en los sitios web para notificar a los usuarios sobre el uso de cookies y obtener su consentimiento según lo establecido por leyes de privacidad como el GDPR (Reglamento General de Protección de Datos) en Europa o la Ley de Cookies en España.

📋 ¿Para qué sirve?

  • Cumplimiento legal: Es obligatorio por ley en muchos países informar sobre el uso de cookies.
  • Transparencia: Informa a los usuarios qué datos se recopilan y con qué finalidad.
  • Consentimiento: Permite a los usuarios aceptar o rechazar el uso de cookies.
  • Personalización: Las cookies permiten recordar preferencias del usuario.
  • Analítica: Ayuda a los propietarios del sitio a entender el comportamiento de los visitantes.

🏗️ Tipos de cookies

  • Cookies técnicas: Necesarias para el funcionamiento básico del sitio.
  • Cookies de análisis: Recopilan información anónima sobre el uso del sitio.
  • Cookies de personalización: Recuerdan preferencias del usuario.
  • Cookies publicitarias: Rastrean hábitos de navegación para mostrar anuncios relevantes.

⚙️ Elementos clave de un footer de cookies

  • Título claro: "Aviso de cookies" o "Este sitio utiliza cookies".
  • Descripción breve: Explicación simple del uso de cookies.
  • Botón de aceptar: Para dar consentimiento.
  • Botón de rechazar/configurar: Para opciones avanzadas (recomendado).
  • Enlace a política de cookies: Para información detallada.
  • Diseño no intrusivo: Que no bloquee el contenido principal.

💡 Buenas prácticas

  • Ser claro y conciso: Explicar qué son las cookies sin tecnicismos.
  • Ofrecer opciones: Permitir aceptar, rechazar o configurar.
  • No ser intrusivo: Que no ocupe demasiado espacio ni moleste la navegación.
  • Diseño responsive: Que se vea bien en todos los dispositivos.
  • No requerir acción forzosa: En algunos países, el consentimiento debe ser activo.
  • Guardar preferencias: Recordar la elección del usuario.

🎯 Alineación y Centrados Complejos con Flexbox

Flexbox no solo sirve para alineaciones básicas, sino que permite resolver problemas de centrado complejos que antes requerían trucos complicados o cálculos manuales. Con las propiedades adecuadas, podemos centrar elementos tanto horizontal como verticalmente, incluso en situaciones con múltiples elementos, tamaños variables o contenido dinámico.

📋 ¿Para qué sirve?

  • Centrado perfecto: Tanto horizontal como vertical, incluso con elementos de tamaño variable.
  • Distribución inteligente: Controlar espacios entre elementos de forma automática.
  • Diseños responsivos: Que se adaptan sin necesidad de media queries complejas.
  • Alineaciones asimétricas: Posicionar elementos en lugares específicos del contenedor.
  • Elementos flotantes: Crear efectos como "sticky" o elementos que se mantienen en posición.

El "Santo Grial" del centrado con Flexbox: centrar un elemento tanto horizontal como verticalmente.
